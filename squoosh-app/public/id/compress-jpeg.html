Mengapa kompres JPEG di browser?

JPEG adalah format output default untuk setiap ponsel, kamera mirrorless, dan alat screenshot. Format sudah lossy, tetapi sebagian besar kamera dan perangkat lunak mengekspor pada kualitas 90+ dengan metadata tambahan (EXIF, XMP, profil ICC) yang membengkakkan file sebesar 20–100 KB masing-masing. Mengenkode ulang dengan encoder modern seperti mozjpeg menghilangkan overhead tersebut dan menggunakan tabel kuantisasi yang lebih baik, sehingga kualitas visual yang sama membutuhkan lebih sedikit byte.

Pixshrink menjalankan mozjpeg sebagai WebAssembly langsung di tab Anda. File yang Anda pilih tidak pernah meninggalkan perangkat Anda — tanpa upload, tanpa antrian server, tanpa dinding pendaftaran. Anda dapat mengompresi folder foto pernikahan dalam pesawat, atau memperkecil 200 foto produk untuk Shopify, dan tidak ada yang dicatat di mana pun.

Berapa banyak reduksi ukuran yang harus saya harapkan?

Hasil tergantung pada sumber, tetapi pada foto ponsel 4032×3024 yang khas pada kualitas 95 asli, pipeline mozjpeg pada kualitas 75 menghasilkan file 80–90% lebih kecil tanpa perbedaan visual dalam tampilan berdampingan. Jika Anda mendorong ke kualitas 60, Anda bisa mencapai reduksi 95%; di bawah itu, blok DCT 8×8 JPEG mulai muncul sebagai ringing di sekitar tepi dengan kontras tinggi.

Slider adalah kontrol utama. Beberapa aturan praktis: kualitas 80 adalah sweet spot untuk foto hero di situs web; kualitas 70 cocok untuk thumbnail produk dan avatar; kualitas 50 berfungsi untuk gambar pratinjau yang dimuat sebelum versi resolusi lebih tinggi.

Bagaimana dengan EXIF dan metadata?

Secara default, Pixshrink menghapus EXIF (kamera, GPS, timestamp) dari JPEG. Langkah ini saja biasanya menghemat 30–80 KB dan menghilangkan efek samping privasi dari pengiriman foto dengan alamat rumah Anda terpatri di file. Anda dapat mengaktifkan kembali preservasi metadata dari panel opsi jika Anda membutuhkannya untuk pekerjaan arsip.

Coba juga format lain

Jika Anda ingin file sekecil mungkin dengan kualitas visual yang sama, mengenkode ulang ke WebP atau AVIF biasanya mengalahkan JPEG sebesar 20–40% tambahan. Untuk screenshot dan tangkapan UI dengan teks tajam dan warna datar, PNG seringkali sudah optimal dan dapat diperkecil tanpa loss dengan oxipng. Untuk animasi, lihat kompresi GIF.

Pertanyaan yang sering diajukan

Apakah kompresor JPEG ini benar-benar gratis?

Ya — tanpa pendaftaran, tanpa masa coba, tanpa batas harian, tanpa watermark. Encoder berjalan di browser Anda, jadi tidak ada biaya server untuk diteruskan.

Apakah foto saya diupload ke suatu tempat?

Tidak. File dibaca ke memori, diproses oleh mozjpeg yang dikompilasi ke WebAssembly, dan hasilnya ditawarkan sebagai unduhan. Aktivitas jaringan selama kompresi adalah nol.

Apakah kompresi mengurangi kualitas gambar?

JPEG lossy secara desain, jadi setiap pengenkodean ulang kehilangan sebagian informasi. Slider kualitas mengontrol seberapa banyak. Pada 75–80 kerugian tidak terlihat oleh kebanyakan penonton; di bawah 50 Anda mungkin melihat ringing dan banding pada gradien halus.

Bisakah saya mengompres banyak JPEG sekaligus secara batch?

Ya. Jatuhkan folder ke halaman, sesuaikan kualitas, dan Pixshrink akan mengenkode semuanya dan mengunduh satu ZIP. Tidak ada batas jumlah file, tetapi batch yang sangat besar dibatasi oleh RAM yang tersedia.
