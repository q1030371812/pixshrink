AV1 코덱을 감싸는 libavif 래퍼로 AVIF 파일을 재인코딩하거나 재압축하세요. 일치된 인지 품질에서 AVIF는 일반적으로 WebP보다 20~30% 더 작고 JPEG보다 50% 이상 더 작습니다.

AVIF가 무엇이고 왜 신경 써야 합니까?

AVIF(AV1 Image File Format)는 AV1 비디오 코덱 주변의 정지 이미지 컨테이너입니다. 처음부터 웹에서 JPEG과 WebP를 대체하도록 설계되었습니다: 더 나은 인트라 예측, 더 큰 블록 변환, 필름 그레인 합성, 10비트 및 12비트 색상에 대한 기본 지원. 벤치마크에서 압축 이득은 실재하며 가시적입니다 — Netflix, YouTube 및 대부분의 대규모 콘텐츠 사이트은 이제 디코딩 가능한 브라우저 사용자에게 AVIF를 제공합니다.

Pixshrink는 libavif를 사용하며, 이는 AV1 인코더(기본값 rav1e)를 감쌉니다. 대기업 사이트이 사용하는 것과 동일한 코덱과 깔끔한 브라우저 내 인터페이스를 얻습니다.

실제로 AVIF는 얼마나 더 작습니까?

일반적인 제품 사진 배치에서 JPEG 품질 80에서 AVIF 품질 50으로(SSIM2에서 인지적으로 동등) 전환하면 파일 크기가 대략 50% 절감됩니다. 기본 WebP에서 일치 품질 AVIF로 전환하면 일반적으로 추가로 20~30% 절감됩니다. 랜딩 페이지의 히어로 이미지의 경우 이는 280 KB와 95 KB 로드 간의 차이로, Core Web Vitals 및 Lighthouse 점수에 직접 반영됩니다.

speed 슬라이더는 인코더 사전 설정을 제어합니다. Speed 0이 가장 느리고 가장 작음; Speed 6이 빠르며 약간 더 큼. Speed 4는 일회성 작업의 좋은 기본값; Speed 8은 일괄 처리에 적합합니다.

브라우저 및 도구 지원

AVIF는 Chrome 85+, Firefox 93+, Edge 121+, Safari 16+(2022년 출시)에서 지원됩니다. 글로벌 웹 인구의 커버리지는 이제 95%를 훨씬 넘습니다. 구형 브라우저를 지원해야 하는 경우, 올바른 패턴은 WebP 또는 JPEG 폴백과 함께 <picture> 요소를 제공하는 것입니다 — Pixshrink의 같은 페이지에서 한 번의 업로드로 세 가지 모두를 생성할 수 있습니다.

원할 수 있는 다른 형식

색상 변화 없이 완전한 투명도를 위해 PNG 무손실이 여전히 안전한 선택입니다. 애니메이션의 경우 AVIF는 지원하지만 도구 성숙도가 GIF 및 애니메이션 WebP만큼 높지 않습니다 — 정말 추가 절감이 필요하지 않는 한 애니메이션 WebP부터 시작하세요.

자주 묻는 질문

Pixshrink는 어떤 AV1 인코더를 사용합니까?

기본 AV1 인코더로 rav1e를 사용하는 libavif입니다. 대부분의 프로덕션 서버가 사용하는 것과 동일한 인코더 경로이며 브라우저에서 실행되도록 WebAssembly로 컴파일됩니다.

AVIF가 WebP보다 더 작습니까?

예. 일치된 인지 품질에서 AVIF는 일반적으로 WebP보다 20~30% 더 작고 JPEG보다 약 50% 더 작습니다. 비용은 인코드 시간과 약간 더 제한된 도구 생태계입니다.

어떤 브라우저가 AVIF를 표시할 수 있습니까?

Chrome 85+, Edge(모든 최신 버전), Firefox 93+, Safari 16+, Opera, 그리고 지난 3년간 출시된 모든 모바일 브라우저. 구형 브라우저의 경우 WebP 또는 JPEG 폴백을 사용하세요.

AVIF가 투명도와 애니메이션을 지원합니까?

둘 다 지원합니다. 8비트 및 10비트 알파가 지원되며, AVIF는 여러 프레임을 담을 수 있지만 오늘날 대부분의 사용 사례에서는 애니메이션 WebP가 여전히 더 안전한 선택입니다.
